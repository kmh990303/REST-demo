# RESTful 라우트

* REST <br>
 - 클라이언트와 서버가 어떻게 소통해야 하는가에 대한 가이드라인, 개념, 원리
 - 웹 앱을 설계할 때 따르는 일련의 원칙 (클라이언트 서버 커뮤니케이션을 위함) <br>

* RESTful
 - REST 규칙에 따르는 시스템
 - 리소스: 하나의 엔티티 / ex) 사용자, 메세지 등등

* Get <br>
 - 정보를 가져올 때, 검색할 때 사용
 - 데이터는 쿼리 문자열에 담김
<br>

* Post <br>
 - 정보를 올리거나 보낼 때 사용 <br>

*Patch <br>
 - 기존의 내용을 편집하거나 바꿀 때 쓴다 <br>
